# Advisor Reference Material

Add small, focused Markdown notes here when the army advisor needs more local context.

Guidelines:

- Use one topic per file, with a descriptive filename.
- Summarize doctrine, list-building preferences, and allowed interpretations in your own words.
- Do not paste secrets, API keys, or large copyrighted rulebook sections.
- Keep each file short. The backend loads files in filename order and caps total reference text.
- Run the advisor context and API tests after changes.
